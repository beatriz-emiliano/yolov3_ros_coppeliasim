Regular API function

simIsHandle / sim.isHandle

Description Checks whether a general object handle is still valid. When a general object is destroyed (e.g. programmatically or via the user interface), then its related handle is not valid anymore and will trigger an error when used. Use this function to avoid triggering an error. See also sim.getObject.
C/C++
synopsis 		simInt simIsHandle(simInt objectHandle,simInt reserved=-1)
C/C++
parameters
objectHandle: handle of the object
reserved: set to -1
C/C++
return value
-1 if operation was not successful, 0 if the handle is not valid anymore, or 1 if the handle is still valid.
